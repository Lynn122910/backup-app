## 1. 系统架构总览

### 1.1 分层架构

本系统采用分层 + 模块化架构，分为四层：

```
┌─────────────────────────────────────────────────────────────┐
│                     GUI 表现层 (Qt)                           │
│  MainWindow | 各功能面板 | 进度对话框 | 日志查看器 | 系统托盘   │
├─────────────────────────────────────────────────────────────┤
│                    业务逻辑层 (Core)                          │
│  BackupEngine | RestoreEngine | TaskScheduler | ConfigMgr   │
├─────────────────────────────────────────────────────────────┤
│                    功能模块层 (Services)                      │
│  FileScanner | FilterEngine | Compressor | Encryptor         │
│  MetadataHandler | IncrementalStrategy | ArchiveFormat       │
├─────────────────────────────────────────────────────────────┤
│                    基础设施层 (Infra)                          │
│  Logger | ErrorHandler | FileUtils | PlatformUtils | Thread  │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 核心技术选型

| 技术项 | 选型 | 理由 |
|--------|------|------|
| 编程语言 | C++17 | 性能优异，QT 原生支持，与 Linux 系统调用深度集成 |
| GUI 框架 | Qt 5.15+ Widgets | 成熟稳定，跨平台，组件丰富 |
| 构建工具 | CMake 3.16+ | 跨平台标准，与 Qt 集成良好 |
| 单元测试 | Google Test (gtest) | C++ 测试标准 |
| 压缩库 | zlib + libzstd | zlib 标准兼容性好，zstd 压缩性能卓越 |

| JSON 解析 | nlohmann/json (header-only) | 轻量、易用、C++ 原生 |
| 日志库 | spdlog | 高性能、header-only、格式丰富 |

| 性能分析 | gprof / perf | Linux 性能分析标准 |

### 1.3 第三方库依赖策略

遵循实验要求：**第三方库仅用于底层算法支持（压缩、加密），不得直接提供备份业务功能**。

| 库 | 用途 | 是否合规 | 说明 |
|----|------|----------|------|
| zlib / libzstd | 压缩算法 | ✅ 合规 | 算法库，不涉及备份业务 |
| OpenSSL | 加密/哈希算法 | ✅ 合规 | 算法库，不涉及备份业务 |
| nlohmann/json | JSON 解析 | ✅ 合规 | 数据格式解析 |
| spdlog | 日志 | ✅ 合规 | 通用日志 |
| gtest | 测试 | ✅ 合规 | 测试框架 |
| Qt | GUI + 基础设施 | ✅ 合规 | 明确要求使用 |

## 3. 核心模块设计

### 3.1 公共类型定义（common/types.h）

```cpp
#pragma once

#include <string>
#include <vector>
#include <cstdint>
#include <ctime>
#include <optional>
#include <nlohmann/json.hpp>

namespace backup {

/// 文件类型枚举
enum class FileType {
    kRegular     = 0,   // 普通文件
    kDirectory   = 1,   // 目录
    kSymlink     = 2,   // 符号链接
    kHardLink    = 3,   // 硬链接
    kFifo        = 4,   // 命名管道
    kBlockDevice = 5,   // 块设备
    kCharDevice  = 6,   // 字符设备
    kSocket      = 7,   // 套接字
    kUnknown     = 99,
};

/// 文件元数据
struct FileMetadata {
    std::string path;              // 相对于源目录的路径
    FileType    type;              // 文件类型
    uint64_t    size;              // 文件大小（字节）
    uint32_t    mode;              // 权限 mode_t
    uint32_t    uid;               // 所有者 UID
    std::string owner;             // 所有者用户名
    uint32_t    gid;               // 所属组 GID
    std::string group;             // 所属组名
    int64_t     mtime_nsec;        // 修改时间（纳秒）
    int64_t     atime_nsec;        // 访问时间（纳秒）
    int64_t     ctime_nsec;        // 元数据变更时间（纳秒）
    uint64_t    inode;             // inode 号（硬链接检测用）
    uint32_t    dev_major;         // 设备主号
    uint32_t    dev_minor;         // 设备次号
    std::string symlink_target;    // 符号链接目标
    std::string checksum;          // SHA-256 校验和
    std::map<std::string, std::string> xattrs;  // 扩展属性
    std::vector<std::string>       acl;         // ACL 条目

    /// JSON 序列化
    nlohmann::json ToJson() const;
    static std::optional<FileMetadata> FromJson(const nlohmann::json& j);
};

/// 备份文件条目（用于清单中的文件记录）
struct BackupFileEntry {
    FileMetadata metadata;
    std::string  status;            // "added" | "modified" | "deleted" | "unchanged"
    uint64_t     offset_in_archive; // 在归档文件中的偏移
    uint64_t     compressed_size;   // 压缩后在归档中的大小
};

/// 备份类型
enum class BackupType {
    kFull         = 0,   // 全量备份
    kIncremental  = 1,   // 增量备份（相对于上一次备份）
    kDifferential = 2,   // 差异备份（相对于上一次全量备份）
};

/// 压缩算法
enum class CompressionAlgo {
    kNone   = 0,
    kZlib   = 1,
    kZstd   = 2,
};

/// 加密算法
enum class EncryptionAlgo {
    kNone       = 0,
    kAes256Gcm  = 1,
};

/// 筛选规则运算符
enum class FilterOp {
    kInclude = 0,   // 包含匹配
    kExclude = 1,   // 排除匹配
};

/// 淘汰策略类型
enum class RetentionType {
    kByCount = 0,   // 按数量
    kByAge   = 1,   // 按时间
    kBySize  = 2,   // 按空间
};

} // namespace backup
```

### 3.2 文件扫描器（FileScanner）

**职责**：递归遍历目录，收集文件信息，处理特殊文件类型。

```cpp
// file_scanner.h
#pragma once

#include "common/types.h"
#include <string>
#include <vector>
#include <functional>

namespace backup {

/// 进度回调：current / total
using ProgressCallback = std::function<void(uint64_t current, uint64_t total)>;

/// 文件扫描器
class FileScanner {
public:
    FileScanner() = default;
    ~FileScanner() = default;

    /// 扫描目录，返回所有文件信息列表
    /// @param root_path 根目录路径
    /// @param recursive 是否递归
    /// @param progress_cb 进度回调
    /// @return 文件元数据列表
    std::vector<FileMetadata> Scan(
        const std::string& root_path,
        bool recursive = true,
        ProgressCallback progress_cb = nullptr);

    /// 扫描并统计文件数量（不含目录）
    uint64_t CountFiles(const std::string& root_path, bool recursive = true);

    /// 扫描并计算总大小
    uint64_t TotalSize(const std::string& root_path, bool recursive = true);

private:
    /// 获取单个文件的元数据
    FileMetadata GatherMetadata(const std::string& full_path,
                                const std::string& relative_path,
                                const struct stat& st);

    /// 读取扩展属性
    std::map<std::string, std::string> ReadXattrs(const std::string& path);

    /// 读取 ACL
    std::vector<std::string> ReadAcl(const std::string& path);
};

} // namespace backup
```

**实现要点**：
- 使用 `nftw()` 或 `readdir()` + 递归进行目录遍历
- 使用 `lstat()` 获取文件信息（不跟随符号链接）
- 通过 `S_ISREG/S_ISDIR/S_ISLNK/S_ISFIFO/S_ISBLK/S_ISCHR/S_ISSOCK` 宏判断文件类型
- 通过 `fgetxattr()` / `listxattr()` 读取扩展属性
- 通过 `acl_get_file()` 读取 ACL

### 3.3 筛选引擎（FilterEngine）

**职责**：根据用户配置的筛选规则，判断文件是否应被备份。

```cpp
// filter_engine.h
#pragma once

#include "common/types.h"
#include <string>
#include <vector>
#include <regex>
#include <optional>

namespace backup {

/// 单条筛选规则
struct FilterRule {
    FilterOp            op;           // include 或 exclude
    std::string         description;  // 规则描述

    // 匹配条件（均为可选，未设置表示不限制）
    std::optional<std::string>      path_glob;      // 路径 glob 模式
    std::optional<std::string>      name_regex;     // 文件名正则
    std::optional<std::vector<FileType>> file_types; // 文件类型
    std::optional<int64_t>          mtime_after;    // 修改时间之后
    std::optional<int64_t>          mtime_before;   // 修改时间之前
    std::optional<uint64_t>         min_size;       // 最小大小
    std::optional<uint64_t>         max_size;       // 最大大小
    std::optional<std::string>      owner;          // 所有者
    std::optional<std::string>      group;          // 所属组

    nlohmann::json ToJson() const;
    static FilterRule FromJson(const nlohmann::json& j);
};

/// 筛选引擎
class FilterEngine {
public:
    FilterEngine() = default;

    /// 添加一条筛选规则
    void AddRule(const FilterRule& rule);

    /// 清空所有规则
    void Clear();

    /// 判断文件是否应该被包含（经过所有规则筛选后）
    /// @return true = 需要备份，false = 跳过
    bool ShouldInclude(const FileMetadata& meta) const;

    /// 批量筛选
    std::vector<FileMetadata> Filter(
        const std::vector<FileMetadata>& files,
        std::function<void(size_t, size_t)> progress_cb = nullptr) const;

private:
    /// 单条规则是否匹配
    bool RuleMatches(const FilterRule& rule, const FileMetadata& meta) const;

    /// Glob 匹配
    bool MatchGlob(const std::string& pattern, const std::string& path) const;

    std::vector<FilterRule> rules_;
};

} // namespace backup
```

**筛选逻辑流程**：
```
对于每个文件:
    1. 先检查所有 exclude 规则，若任一匹配 → 跳过
    2. 再检查所有 include 规则，若至少有一条匹配 → 保留
    3. 若无 include 规则 → 默认保留
    4. 若有 include 规则但均不匹配 → 跳过
```

### 3.5 压缩模块（Compressor）

**设计模式**：策略模式 + 工厂模式

```cpp
// compressor.h
#pragma once

#include "common/types.h"
#include <vector>
#include <memory>
#include <cstdint>

namespace backup {

/// 压缩器抽象接口
class Compressor {
public:
    virtual ~Compressor() = default;

    /// 算法名称
    virtual std::string AlgorithmName() const = 0;

    /// 压缩数据
    /// @param input 原始数据
    /// @param level 压缩级别 (0-9)
    /// @return 压缩后数据
    virtual std::vector<uint8_t> Compress(const std::vector<uint8_t>& input,
                                          int level = 3) = 0;

    /// 解压数据
    /// @param input 压缩数据
    /// @param original_size 原始大小（某些算法需要）
    /// @return 解压后数据
    virtual std::vector<uint8_t> Decompress(const std::vector<uint8_t>& input,
                                            uint64_t original_size = 0) = 0;

    /// 流式压缩：压缩一个数据块（用于大文件分块压缩）
    virtual std::vector<uint8_t> CompressChunk(const std::vector<uint8_t>& chunk,
                                               bool is_last = false) = 0;

    /// 流式解压：解压一个数据块
    virtual std::vector<uint8_t> DecompressChunk(const std::vector<uint8_t>& chunk,
                                                 bool is_last = false) = 0;

    /// 重置流状态
    virtual void Reset() = 0;
};

/// Zlib 压缩实现
class ZlibCompressor : public Compressor {
public:
    std::string AlgorithmName() const override { return "zlib"; }
    std::vector<uint8_t> Compress(const std::vector<uint8_t>& input, int level) override;
    std::vector<uint8_t> Decompress(const std::vector<uint8_t>& input, uint64_t) override;
    std::vector<uint8_t> CompressChunk(const std::vector<uint8_t>& chunk, bool is_last) override;
    std::vector<uint8_t> DecompressChunk(const std::vector<uint8_t>& chunk, bool is_last) override;
    void Reset() override;
private:
    // z_stream 状态在此管理
    void* z_stream_;  // 不暴露 zlib 头文件到接口
};

/// Zstd 压缩实现
class ZstdCompressor : public Compressor {
    // ... 类似实现
};

/// 压缩器工厂
class CompressorFactory {
public:
    static std::unique_ptr<Compressor> Create(CompressionAlgo algo);
    static std::unique_ptr<Compressor> Create(const std::string& name);
};

} // namespace backup
```

### 3.6 加密模块（Encryptor）

```cpp
// encryptor.h
#pragma once

#include "common/types.h"
#include <vector>
#include <memory>
#include <cstdint>
#include <string>

namespace backup {

/// 加密密钥（派生后的密钥）
struct EncryptionKey {
    std::vector<uint8_t> key;       // 加密密钥
    std::vector<uint8_t> salt;      // 盐值
    std::vector<uint8_t> iv;        // IV / Nonce
    uint32_t             iterations; // KDF 迭代次数
};

/// 加密器抽象接口
class Encryptor {
public:
    virtual ~Encryptor() = default;

    /// 算法名称
    virtual std::string AlgorithmName() const = 0;

    /// 从密码派生密钥
    /// @param password 用户密码
    /// @param salt 盐值（为空则随机生成）
    /// @return 派生后的密钥材料
    virtual EncryptionKey DeriveKey(const std::string& password,
                                    const std::vector<uint8_t>& salt = {}) = 0;

    /// 加密数据
    /// @param plaintext 明文
    /// @param key 密钥材料
    /// @return 密文（含认证标签）
    virtual std::vector<uint8_t> Encrypt(const std::vector<uint8_t>& plaintext,
                                         const EncryptionKey& key) = 0;

    /// 解密数据
    /// @param ciphertext 密文（含认证标签）
    /// @param key 密钥材料
    /// @return 明文（若认证失败返回空）
    virtual std::optional<std::vector<uint8_t>> Decrypt(
        const std::vector<uint8_t>& ciphertext,
        const EncryptionKey& key) = 0;
};

/// AES-256-GCM 加密实现
class Aes256GcmEncryptor : public Encryptor {
public:
    std::string AlgorithmName() const override { return "AES-256-GCM"; }
    EncryptionKey DeriveKey(const std::string& password,
                            const std::vector<uint8_t>& salt) override;
    std::vector<uint8_t> Encrypt(const std::vector<uint8_t>& plaintext,
                                 const EncryptionKey& key) override;
    std::optional<std::vector<uint8_t>> Decrypt(
        const std::vector<uint8_t>& ciphertext,
        const EncryptionKey& key) override;
};

/// 加密器工厂
class EncryptorFactory {
public:
    static std::unique_ptr<Encryptor> Create(EncryptionAlgo algo);
};

} // namespace backup
```

**密钥派生详情**：
```
PBKDF2-HMAC-SHA256:
  password + salt (随机 16 字节) → 100,000 次迭代 → 256 位 AES 密钥

AES-256-GCM:
  随机 12 字节 nonce → Encrypt(plaintext) → ciphertext + 16 字节认证标签
```

```cpp
// backup_manifest.h
#pragma once

#include "common/types.h"

namespace backup {

class BackupManifest {
public:
    BackupManifest();

    // ========== 属性 ==========

    std::string backup_id() const;
```

**备份引擎执行流程**：

```
1. 校验参数
2. 创建清单对象
3. 扫描源目录 → 获取文件列表
4. 应用筛选规则 → 过滤文件列表
5. 如果是增量备份：
   a. 读取上次清单
   b. 使用增量策略对比变更
   c. 仅保留变更文件
6. 计算各文件 SHA-256 校验和
7. 创建归档写入器
8. 配置压缩器/加密器（如启用）
9. 逐个文件：读取 → 压缩 → 加密 → 写入归档
10. 写入清单到归档
11. 写入归档尾部
12. 保存独立的清单文件（便于快速查询）
13. 完成
```


```
│                    UI 线程 (Main Thread)               │
│  - 事件循环                                           │
│  - 界面更新                                           │

│  - BackupEngine::Execute()                           │
```

**关键原则**：
- 备份/还原操作在独立的 `QThread` 中执行，不阻塞 UI
- 进度信息通过 `Qt::QueuedConnection` 传递到 UI 线程
- 取消操作通过 `std::atomic<bool>` 标志位实现线程安全中断
- 所有对归档文件的读写操作在工作线程中完成

### 7.1 错误码体系

```cpp
// common/error_codes.h
namespace backup {

enum class ErrorCode : int {
    kSuccess                = 0,

    // 参数错误 (1xx)

    kSourceNotFound         = 102,

    kFileReadError          = 204,
    kFileWriteError         = 205,
```

### 7.2 异常安全保证

- **核心数据操作**：基本保证（Basic Guarantee）——操作失败时数据状态有效但不一定完整
- **配置读写**：强保证（Strong Guarantee）——先写入临时文件，成功后再原子替换
- **归档写入**：通过尾部校验和提供可检测的完整性

## 8. 关键技术难点与解决方案

### 8.1 硬链接处理

**难点**：备份时如何避免硬链接指向的同一 inode 被重复存储。

**方案**：
1. 扫描时记录每个文件的 `(dev, inode)` 对
2. 遇到相同 `(dev, inode)` 时，仅存储第一个实例的数据
3. 在清单中记录硬链接关系：`hard_link_to: "path/of/first/instance"`
4. 还原时：先提取第一个实例，然后对后续硬链接条目调用 `link()` 创建硬链接

### 8.2 大文件流式处理

**难点**：GB 级文件无法一次性读入内存。

**方案**：
- 使用 1MB 分块读取源文件
- 每块独立压缩/加密后流式写入归档
- `Compressor` 接口提供 `CompressChunk/DecompressChunk` 方法维护内部流状态
- 内存峰值控制在 `chunk_size * 缓冲系数` 范围内

### 8.3 压缩与加密的顺序

**原则**：先压缩，再加密。

**理由**：
- 压缩依赖于数据的冗余性，加密后数据看起来是随机的，无法有效压缩
- 加密在压缩之后，仍能保护数据机密性

**流程**：`原始数据 → 压缩 → 加密 → 写入归档`

### 8.4 备份中断与恢复

**方案**：
- 在备份过程中周期性写入检查点（checkpoint）信息到归档文件
- 检查点记录：已备份文件索引、归档文件偏移
- 中断后重启时，读取最后一个有效检查点，跳过已完成文件继续备份
- 清单在全部完成后才写入，未完成的归档文件不含清单视为不完整
