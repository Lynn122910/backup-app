# 筛选功能测试说明

## 准备工作

1. 先生成测试文件：
```bash
cd ~/Desktop
bash generate_test_files.sh ~/backup_test_source
```

2. 启动备份应用，在 GUI 中：
   - 源目录选择：`/home/ubuntu/backup_test_source`
   - 目标目录选择：`/home/ubuntu/Desktop`

## 测试 1：按名称筛选（name_regex）

**目的**：只备份 `.txt` 文件，排除其他扩展名。

**操作**：
1. 点击「Filter Rules...」→「Add...」
2. 填写：
   - Action: **Include (keep matching files)**
   - Description: `只备份 txt 文件`
   - Name match (regex): `\.txt$`
   - 其余留空
3. 点击 OK → OK
4. 点击「Start Backup」

**预期结果**：
- 日志显示 `Filters applied: X of 22 files kept`，保留数应约为 8（含 readme.txt、old_archive.txt、recent.txt、today.txt 等）
- 备份中**不应**包含 `app.log`、`config.ini`、`data.bin` 等

---

## 测试 2：按路径筛选（path_glob）

**目的**：只备份 `subdir` 目录下的文件。

**操作**：
1. 先删除之前的规则（选中 → Remove）
2. 点击「Add...」
3. 填写：
   - Action: **Include (keep matching files)**
   - Path match (glob): `subdir/*`
   - 其余留空
4. OK → Start Backup

**预期结果**：
- 只备份 `subdir/nested_file.txt`
- 其余文件全部被排除

---

## 测试 3：按文件类型筛选（file_types）

**目的**：排除所有特殊文件类型（链接、管道、设备等），只保留普通文件和目录。

**操作**：
1. 清空旧规则，添加新规则
2. 填写：
   - Action: **Exclude (remove matching files)**
   - Description: `排除特殊文件` 
   - File types: 勾选 **Symlink、Hardlink、FIFO、Block、Char、Socket**（不勾 Regular 和 Dir）
   - 其余留空
3. OK → Start Backup

**预期结果**：
- 备份中**不应**包含 `link_to_readme`、`hardlink_readme`、`myfifo`、`test_socket`、`char_null`、`block_loop0`
- 普通文件和目录正常保留

---

## 测试 4：按时间筛选（mtime_after / mtime_before）

**目的**：只备份 2024-06-01 之后修改的文件。

**操作**：
1. 清空旧规则，添加新规则
2. 填写：
   - Action: **Include (keep matching files)**
   - Description: `只备份近期文件`
   - Modification time range: 勾选「Modified after」，日期选 **2024-06-01 00:00:00**
   - 其余留空
3. OK → Start Backup

**预期结果**：
- `old_archive.txt`（2023-01-15）被排除
- `recent.txt`（2024-07-01）、`today.txt` 等近期文件被保留

---

## 测试 5：组合筛选（Include + Exclude）

**目的**：备份所有 `.txt` 和 `.log` 文件，但排除隐藏文件。

**规则 1**：
- Action: Include
- Name match (regex): `\.(txt|log)$`

**规则 2**：
- Action: Exclude
- Name match (regex): `^\.`

**操作**：
1. 添加规则 1（Include：`.txt` 和 `.log`）
2. 再添加规则 2（Exclude：`.` 开头）
3. Start Backup

**预期结果**：
- 包含 `readme.txt`、`app.log`、`recent.txt` 等 `.txt`/`.log` 文件
- 排除 `.hidden_config`（被规则 2 排除）
- 排除 `config.ini`、`data.bin` 等（未匹配规则 1）

---

## 验证方法

每次备份完成后，检查备份归档中实际包含了哪些文件：

```bash
# 查看备份文件列表（manifest JSON）
cat /home/ubuntu/Desktop/*.manifest.json | python3 -m json.tool 2>/dev/null | grep '"path"'
```

对比实际文件列表与预期结果是否一致。

## 故障排查

| 问题 | 可能原因 |
|------|----------|
| 所有文件都被排除了 | Include 规则的 regex/glob 写错了 |
| 筛选没生效 | 规则添加后没点 OK，或忘了 Start Backup |
| 特殊文件没被排除 | file_types 没勾全（Symlink/FIFO/Block/Char/Socket） |
| 时间筛选不准 | 时区问题，检查 QDateTimeEdit 是否为 UTC |
