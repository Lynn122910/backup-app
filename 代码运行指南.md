# 数据备份软件 — 代码运行指南

---

## 1. 环境要求

### 1.1 操作系统

- **推荐**: Ubuntu 20.04+ / Debian 11+ / CentOS 7+ / Fedora 36+
- **架构**: x86-64 (amd64)

### 1.2 编译工具链

| 工具 | 最低版本 | 说明 |
|------|----------|------|
| GCC | 7.5+ (推荐 9+) | C++17 支持 |
| CMake | 3.16+ | 构建系统 |
| make | 4.0+ | 构建执行 |
| pkg-config | 0.29+ | 包查找工具 |

### 1.3 依赖库

| 库 | Ubuntu/Debian 包名 | CentOS/Fedora 包名 | 说明 |
|----|--------------------|--------------------|------|
| Qt5 | `qt5-default` (或 `qtbase5-dev`) | `qt5-qtbase-devel` | GUI 框架 |
| OpenSSL | `libssl-dev` | `openssl-devel` | 加密和 SHA-256 |
| zlib | `zlib1g-dev` | `zlib-devel` | 压缩支持 |

## 2. 快速开始

### 2.1 Ubuntu / Debian

```bash
# 步骤 1: 安装系统依赖
sudo apt update
sudo apt install -y build-essential cmake qtbase5-dev \
    libssl-dev zlib1g-dev pkg-config

# 步骤 2: 进入项目目录
cd backup-app

# 步骤 3: 执行构建脚本
chmod +x scripts/build.sh
./scripts/build.sh release

# 步骤 4: 运行程序
./build/src/backup-app
```

### 2.2 CentOS / Fedora

```bash
# 步骤 1: 安装系统依赖
sudo dnf install -y gcc-c++ cmake qt5-qtbase-devel \
    openssl-devel zlib-devel pkgconf

# 步骤 2-4: 同 Ubuntu
cd backup-app
./scripts/build.sh release
./build/src/backup-app
```

### 2.3 手动构建（如果脚本不适用）

```bash
cd backup-app
mkdir -p build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release
make -j$(nproc)
./src/backup-app
```

## 3. 使用说明

### 3.1 创建备份

1. 启动程序后，默认处于 **💾 Backup** 标签页
2. 点击 **Browse...** 选择要备份的源目录
3. 点击 **Browse...** 选择备份文件存放的目标文件夹
4. （可选）输入备份名称
5. 点击 **▶ Start Backup** 开始备份
6. 观察进度条和日志输出
7. 备份完成后弹出提示对话框

**备份产物**：
- `<目标目录>/backup_YYYYMMDD_HHMMSS_xxx.bkp` — 备份归档文件
- `<目标目录>/backup_YYYYMMDD_HHMMSS_xxx.bkp.manifest.json` — 备份清单（JSON 格式，可人工查看）

### 3.2 还原备份

1. 切换到 **📂 Restore** 标签页
2. 点击 **Browse...** 选择 `.bkp` 备份归档文件
3. 点击 **Browse...** 选择还原目标目录
4. 点击 **▶ Start Restore** 开始还原
5. 等待还原完成

### 3.3 查看备份内容（不还原）

备份清单文件（`.manifest.json`）是纯文本 JSON 格式，可以用任何文本编辑器打开查看备份中包含哪些文件、文件元数据等信息。

## 6. 常见问题

### Q1: CMake 找不到 Qt5

```bash
# Ubuntu/Debian
sudo apt install qtbase5-dev

# 或者使用 Qt6
sudo apt install qt6-base-dev
# 然后修改 CMakeLists.txt 中的 Qt 查找顺序
```

### Q2: 编译时报 "SHA256_Init" 未定义

需要链接 OpenSSL：
```bash
sudo apt install libssl-dev
```

### Q3: 提示 "Failed to create archive file"

检查目标目录是否有写入权限。

### Q4: 备份很慢

正常。大量小文件备份受磁盘 I/O 限制。大文件使用 1MB 分块流式读写。

### Q5: 还原后文件权限不对

以非 root 用户运行时，无法还原文件的所有者（owner/group）信息。权限（mode）和时间戳可以正常还原。

## 7. 归档格式说明

`.bkp` 文件是自定义的二进制归档格式：

```
[Header]  4B magic "BKP0" + 2B version + 2B flags
[Manifest] 8B size + JSON data
[Files]    重复: 2B path_len + path + 1B type + 4B mode + 8B orig_size + 8B stored_size + data
[Footer]   8B file_count + 32B SHA-256 checksum
```

可以使用 `xxd` 或 `hexdump` 查看归档文件头部：
```bash
xxd backup_xxx.bkp | head -20
```

清单 JSON 可以从归档中单独提取（可通过修改源码实现导出工具）。
